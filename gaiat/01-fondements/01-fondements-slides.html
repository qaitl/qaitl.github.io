Section 1

01

Panorama de l'IA

D'où vient-on ?

Quatre grandes époques

1950 — 1980

IA symbolique

Règles écrites par des experts. Si X alors Y. Systèmes experts, arbres de décision.

1990 — 2010

Machine Learning

L'algorithme apprend les patterns à partir d'exemples. Filtres anti-spam.

2010 — 2020

Deep Learning

Réseaux de neurones profonds. Reconnaissance d'images, traduction automatique, Siri / Alexa.

Depuis 2020

IA générative

L'IA ne classe plus, elle crée. Texte, images, code, musique. LLMs au cœur.

À retenir

L'IA générative (generative AI) n'est pas née avec ChatGPT : c'est la 4ᵉ grande vague, qui s'appuie sur toutes les précédentes.

Insister sur la continuité : chaque époque a résolu un problème que l'IA précédente ne savait pas traiter. La GenAI hérite du deep learning.

Section 2

02

Comment fonctionne un LLM ?

Le principe en une phrase

Un grand modèle de langage (grand modèle de langage / large language model, LLM) est un programme qui prédit le mot le plus probable qui vient ensuite.

Jeu de complétion

« Le chat est sur le… » → votre cerveau anticipe tapis ou canapé. Le LLM fait la même chose, mais à une échelle qui donne l'illusion de comprendre.

Ce qui le rend puissant

Volume — milliards de phrases, dizaines de langues
Taille — milliards de paramètres
Instruction-following — affiné (fine-tuning) pour suivre des instructions

Analogie de l'autocomplétion poussée à son paroxysme. Le LLM n'a pas de modèle du monde, il a un modèle du langage.

Ce qu'un LLM ne fait pas

Il ne comprend pas

Pas de sens au sens humain. Il manipule des distributions de probabilités.

Il ne raisonne pas

Il simule un raisonnement plausible, ce qui n'est pas la même chose.

Il n'a pas de mémoire

Aucune persistance entre conversations, sauf configuration spécifique.

Conséquence pour le testeur : ne jamais faire confiance aveuglément à un LLM pour vérifier un fait, compter, ou raisonner précisément — il faut valider.
Cette slide prépare la section "angles morts du LLM" de l'activité 5.

Section 4

04

Les 3 visages de la GenAI

Chatbot · Open source · Outil intégré

1 — Le chatbot grand public

ChatGPT

Gemini

Claude.ai

Avantages

Sans installation, souvent gratuit. Idéal pour apprendre, rédiger, poser des questions générales.

Limites

Messages envoyés vers des serveurs distants. Aucune connaissance du contexte métier de l'entreprise.

2 — L'open source auto-hébergé

Llama 3 + Ollama

Mistral 7B local

Avantages

Le modèle tourne sur votre machine / serveur. Les données ne quittent jamais votre environnement — idéal pour données sensibles (banque, santé, défense).

Limites

Configuration technique nécessaire. Modèles souvent moins puissants à usage général — mais suffisants pour de nombreux cas.

3 — L'outil intégrant un LLM

OutilCe que l'IA apporte
GitHub CopilotGénération de code et de tests unitaires (test unitaire / unit test) dans l'éditeur
Xray AI (Jira)Génération de cas de test (cas de test / test case) depuis une user story
Atlassian IntelligenceRésumé de tickets, rédaction de doc, assistance recherche
Microsoft 365 CopilotRédaction d'emails, résumé de réunions, génération de présentations
Zephyr Scale AISuggestion de cas de test basée sur les exigences
Analogie : un chatbot, c'est un consultant généraliste que vous appelez. Un outil intégré, c'est ce consultant assis à côté de vous, avec accès à vos fichiers.

Usages textuels

Ce que vous pouvez faire aujourd'hui, sans coder

Cinq familles d'usage

Générer

Email pro, 1ʳᵉ version de cas de test (test case), données de test (test data) fictives.

Résumer & reformuler

Specs longues en 10 points, rapport de défaut (defect report) simplifié, jargon juridique clarifié.

Traduire

Doc technique multilingue, adaptation expert → débutant.

Q & A

Expliquer un concept (test exploratoire / exploratory testing), répondre sur un document fourni.

Débogage & analyse

Expliquer un message d'erreur, suggérer des causes possibles, comparer deux versions d'une exigence.

Section 6

06

Activités textuelles

5 exercices pratiques · chatbot gratuit suffisant

Activité 1

Devinette de réponse

Objectif : comprendre que le LLM prédit ce qui est probable, pas ce qui est vrai.

  1. Choisissez une question ambiguë : « Quelle est la meilleure façon de tester un formulaire d'inscription ? »
  2. Chaque participant note en silence 3 idées de réponse.
  3. Envoyez la question, lisez la réponse, comparez.

Ce qu'on apprend : le LLM donne une réponse générique, pas adaptée à votre contexte précis.

Activité 2

Reformule-moi ça

Objectif : explorer la capacité de reformulation selon le public cible.

  1. Prenez un paragraphe technique.
  2. Demandez successivement : à un enfant de 10 ans, à un expert sécurité, en liste à puces.
  3. Comparez les 3 versions.

Ce qu'on apprend : le LLM adapte son registre. Quelle version est « correcte » ?

Activité 3

Compare les chatbots

Objectif : comprendre que les modèles ne sont pas identiques.

  1. Posez la même question à ChatGPT et à Gemini (ou Claude).
  2. Ex. : « Donne-moi 5 cas de test pour la réinitialisation de mot de passe ».
  3. Comparez : identiques ? L'un est-il plus complet, plus prudent ?

Ce qu'on apprend : il n'y a pas un seul « oracle » IA.

Activité 4

Le rapport de défaut amélioré

Objectif : expérimenter l'aide à la rédaction en contexte pro réel.

  1. Rédigez un rapport incomplet : « L'appli plante quand on se connecte parfois ».
  2. Consigne : « Améliore ce rapport : titre, étapes, résultat attendu/obtenu, sévérité ».
  3. Débat : est-ce utilisable ? Que manque-t-il encore ?

Ce qu'on apprend : le LLM structure et complète, mais invente parfois. Validation humaine indispensable.

Activité 5 — Les angles morts du LLM

Objectif : découvrir les tâches triviales qui mettent un LLM en échec.

Pourquoi ces échecs ? Un LLM découpe le texte en tokens (fragments de mots). « tester » peut devenir test + er. Pas de vision caractère par caractère → échec sur les tâches qui l'exigent.
6 épreuves suivent. Chronométrer ~15 min en atelier.

Activité 5 — Les 6 épreuves

1 · Comptage de lettres

« Compte les 't' dans : j'aime tester et trouver des bugs toute la journée »
Réponse : 7. Souvent 5 ou 6, défendus avec assurance.

2 · Inversion de mot

« Inverse les lettres de développement » → tnemelppoleved. Peu de modèles y arrivent du 1ᵉʳ coup.

3 · Comptage de mots répétés

Compter « de » dans une phrase riche en d'. 5 fois. Souvent oubliés.

4 · Classement par longueur

bug · test · régression · acceptance → 3, 4, 10, 10. Inversions fréquentes.

5 · Calendrier simple

1ᵉʳ mars 2024 vendredi → 1ᵉʳ mars 2025 ? Samedi. 366 jours = 52 sem + 2 j.

6 · Logique de texte littéral

« test » apparaît-il dans « les exigences fonctionnelles servent de base à la vérification » ? Non.

Ce qu'on apprend

Excellent pour le sémantique. Peu fiable pour le comptage, les caractères, le calcul exact. Savoir identifier les angles morts = compétence clé du testeur IA.

Multimodal & vision

Au-delà du texte

Qu'est-ce qu'un LLM multimodal ?

Un modèle multimodal (multimodal model) traite et génère plusieurs types de données : texte, image, audio, vidéo, documents.

ModalitéCe que le modèle peut faire
Image — entréeDécrire, répondre à des questions, identifier des anomalies visuelles
Image — sortieGénérer (DALL·E, Midjourney, Stable Diffusion)
Audio — entréeTranscrire et comprendre la parole (Whisper)
Audio — sortieSynthèse vocale, conversations (ChatGPT Voice)
VidéoDécrire, identifier des événements dans une séquence

Les grands modèles multimodaux

GPT-4o

OpenAI

Texte, images, audio en entrée et sortie.

Gemini 1.5 Pro / 2.0 Flash

Google

Texte, images, audio, vidéo, code. Large fenêtre de contexte.

Claude 3.5 / 3.7 Sonnet

Anthropic

Texte et images en entrée, texte en sortie.

Llama 3.2 Vision

Modèle open source multimodal.

Modèles de vision et tests logiciels

Analyse d'UI

Capture d'écran → lister les éléments d'interface (interface element). Comparer maquette et implémentation. Problèmes d'accessibilité (contraste, taille).

Automatisation visuelle

Générer des localisateurs (locator) à partir d'une capture. Analyser les différences de rendu entre navigateurs.

Documentation visuelle

Photo de tableau blanc → transcription automatique en Gherkin ou tableau structuré.

Section 8

08

Activités multimodales

3 exercices pratiques · GPT-4o ou Gemini suffisent

Activité 6

Décris cette interface

Objectif : comprendre ce que le modèle « voit » dans une image.

  1. Prenez une capture d'écran d'une application que vous connaissez.
  2. Consigne : « Liste tous les éléments d'interface : boutons, champs, menus, messages, icônes. »
  3. Comparez avec votre propre inventaire manuel.

Ce qu'on apprend : précis mais manque parfois des détails subtils. Point de départ, pas résultat final.

Activité 7

Trouve la différence

Objectif : simuler un test de non-régression visuelle (visual regression test).

  1. Deux captures « avant / après » de la même page.
  2. Consigne : « Compare ces deux captures et liste les différences. »
  3. Validez : tout détecté ? Fausses différences ?

Ce qu'on apprend : détecte le sémantique (texte, bouton), pas le pixel-diffing rigoureux.

Activité 8

Photo de tableau blanc

Objectif : expérimenter la transcription de contenu non-numérique.

  1. Écrivez 3 à 5 scénarios de test (test scenario) à la main.
  2. Photographiez, puis : « Retranscris en tableau Markdown : ID, Description, Résultat attendu. »
  3. Relisez : erreurs sur l'écriture peu lisible ?

Ce qu'on apprend : gain de temps pour la saisie. Relecture obligatoire.

Raisonnement & IA agentique

Les nouvelles frontières

Les modèles de raisonnement

Depuis fin 2024, une nouvelle génération : les modèles de raisonnement (reasoning model). Avant de répondre, phase de réflexion interne (« chaîne de pensée » visible).

o1, o3

OpenAI

Efficaces sur maths, logique, code.

Claude Sonnet
extended thinking

Anthropic

Raisonnement étendu activable.

Gemini 2.0
Flash Thinking

Google

Raisonnement intégré.

Moins d'erreurs sur les tâches complexes… mais plus lents et plus coûteux.

L'IA agentique

Un agent IA (AI agent) ne se contente pas de répondre : il agit. Il enchaîne plusieurs étapes, utilise des outils externes, adapte son plan.

Outils qu'un agent peut utiliser

  • Naviguer sur le web
  • Exécuter du code en sandbox
  • Lire / écrire des fichiers
  • Appeler des APIs (Jira, Slack, GitHub)
  • Piloter un navigateur (Playwright, Selenium)

Dans le test logiciel

Lire une user story → générer des cas de test (test case) → les pousser dans Jira → exécuter un script Playwright → rapporter. Sans intervention humaine.

Exemples : Devin (Cognition), GitHub Copilot Workspace, Cursor.

Changement de paradigme

On ne demande plus une réponse, on confie une mission. Les capacités ET les risques s'amplifient (actions irréversibles, hallucinations en cascade). La supervision humaine reste essentielle.

Deux façons d'intégrer un LLM

1 · Chatbot direct

« Génère 10 cas de test pour "ajout au panier" »
« Identifie les ambiguïtés de cette user story »
« Explique ce message d'erreur »

Rapide, sans config. Ponctuel.

2 · Application LLM

Outils spécialisés, contexte métier injecté automatiquement : GitHub Copilot, Xray AI, Atlassian Intelligence, Testim, Mabl, M365 Copilot.

Contextuel, intégré, industrialisable.

À retenir

L'IA générative est un accélérateur, pas un remplaçant. Elle déplace l'effort humain vers la validation, le jugement et la créativité — là où la valeur du testeur est la plus forte.

Activité 9

Du brouillon au cas de test

Objectif : chaîne complète exigence → génération → validation.

  1. User story courte : réinitialisation de mot de passe.
  2. Consigne : « Génère une suite : ID, titre, préconditions, étapes, résultat, type (nominal / alternatif / négatif). »
  3. Analyse : cas pertinents ? cas absents (lien expiré, email inexistant) ? Comportements inventés ?

Ce qu'on apprend : base solide rapide ; cas négatifs (negative test case) et limites métier = valeur du testeur.

Activité 10

L'exigence ambiguë

Objectif : utiliser le LLM comme outil d'analyse des exigences (requirements analysis).

  1. Exigence floue : « Le système doit répondre rapidement aux requêtes. »
  2. Consigne : « Identifie les ambiguïtés et reformule de manière testable avec des critères mesurables. »
  3. Comparez à votre interprétation.

Ce qu'on apprend : excellent pour signaler les termes vagues. Le seuil acceptable reste une décision humaine et métier.
