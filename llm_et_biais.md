C'est exactement comme ça que fonctionne un LLM.
Entraîné sur des quantités phénoménales de données — textes, images, code — il prédit le token le plus probable. Pas de compréhension du monde. Pas de raisonnement. Une complétion statistique très, très sophistiquée.
Le modèle se trompe, confabule, et le fait avec une confiance désarmante. Exactement comme je viens de le faire.
Ce qui nous amène à une question inconfortable : qui a choisi les données d'entraînement ? Leur qualité, leur diversité, leur orientation façonnent directement la performance du modèle — mais aussi le fond même de ses réponses. Travail parfois rigoureux. Parfois orienté. Parfois manipulé.
